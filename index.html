Arduino-based Flight Simulator Peripheral?

Today I, with my friends, began to work on the motors in our cars. We’ve learned how to control them with analogue signals and our cars could finally run. However this is not what I wanna talk about right now. It has inspired me to something else.

I’ve been playing Microsoft Flight Simulator 2020 since the day before yesterday. Well, I’d rather say learning it because it was really, really complicated. It was almost impossible to control a Cessna 172S (probably the most easy-to-control aircraft in this game) using mouse and keyboard. Luckily I had an old joystick (older than me!) from my dad, a sidewinder precision 2, which made it a little easier to control my 172s. However, it was far from enough, I still need rudder pedals, a throttle, and many other peripherals for better experience. Sadly, they are costly. I probably cannot buy them, but I have an Arduino Nano, and it receives analogue signals and converts them into digital signals. If there are slide potentiometers which could be connected to arduino, then it should be possible for me to build, at least, a throttle and a mixing ratio controller.

So here is my idea, the components I will need, I think, are:

  1. Two slide potentiometers, which have distance of travel of about 10 centimeters, as analogue inputs for throttle and mixting ratio controller.
  2. An Arduino Leonardo development board. The Nano one I’m using now for my car wouldn’t be suitable because Leonardo uses the USB communication function of ATmega32u4 which allows it to act as keyboards and mouses, while Nano doesn’t. Also, Leonardo supports 12 analogue inputs.
  3. Since there are 12 analogue inputs, other than throttle and mixing ratio controller, I can possibly add a trim wheel or one more throttle in case that I wanna try liners with two engines in the future. I will think about this later, now I am not even sure if the throttle could work or not.
  4. Cases, panels, lights,buttons, anything which would make it look better. I think I can 3-D print these things.

Then I need to write a program for my peripheral. Honestly, I am not sure how to do it. I’ve spent an hour or so to look for solutions on github and some other websites. There were people who made joysticks using Arduino, but it seems that few were interested in throttles and other stuff. The only things I know now, is C++ would be OK for my purpose, and it is quite similar to C, which I’ve been learning these days. Also, communication between the peripheral and my PC would be based on USB-HID protocal. These are all I know now, it is going to be difficult, and I can probably rely on myself only. But I will try my best. After building my car, I’m going to start to work on this.

By the way, Flight Simulator 2020 is really fascinating. Even if you have no interests in flying, it is wonderful to enjoy the scenery around the world in this game. It is going to be the best game of 2020, I think.

OverClock Part II

Several months ago I locked the frequency of my R7 3700X at 4.1 GHz without changing the voltage. Everything was fine until the weather got hotter now. Yesterday after leaving it running for folding@home for hours with the AC in my room off, temperature of the CPU reached 95℃ and my computer shut down automatically. Luckily nothing was broken but I don’t think such accidents are good for its life.

Later when I was trying to reduce its frequency in BIOS I noticed something interesting named Precision Boost Overdrive, which was, according to instructions in my BIOS, a “powerful new feature designed to improve multi-threaded performance” by AMD. Initially I thought it was just another automatic overclocking program but in fact there was a little difference here. The standard frequency was still 3.6 GHz (3.59 GHz according to Windows), but the TDP, TDC and EDC limits were unlocked (still limited by the motherboard, of course) which meant the boost frequency could be maintained longer and higher, especially when few cores were working. The Ryzen Master software showed the maximum boost frequency for single core was increased to 4.6 GHz (I really doubt whether it could really reach that high, though.) .

What’s more, since it was altering the boost frequency, it would not get overheated so easily any more. When I ran folding@home and all 8 cores were processing, the frequency was maintained at 4.05 GHz, which was much lower than that of single-core processing and reduced a lot of heat and kept the computer safe.

Let GPU Engage

After failing to call all the cores of our CPUs to run the process, my teammates and I found another way to increase our speed of training. In the past we were using Tensorflow for CPU, which was easy to install, however there is another version which is speceifically designed for Nvidia GPUs. It was really hard to install the GPU version, we needed to install CUDA, cuDNN, and reinstall an older Python 3.6. One of our teammate gave it up in the end , another one and I made it after retrying several times as the versions of CUDA and cuDNN matters A LOT. Anyway, now our computers could use GPUs to process the images. My RTX2070Super finished the task in 50 minutes, which was more than ten times shorter than the time taken by CPU. This will definitely help us a lot in the competition as we can train our model more times than the other teams.

Multi-thread Processing

In the past several weeks I was participating in a challenge to train a model for automated garbage classification. Together with two of my classmates, I used inception_v3 to process the images. We had a training set which contains 10,000 pictures which required us to run the program for more than 10 hours each time. Initially I thought that it was because of the huge number of image, or maybe the performance of my PC is just not good enough. However, later I noticed that while running the program the CPU fan seemed to be working slowly which means the CPU load is low. After checking Ryzen Master I realized that only two of the eight cores of my CPU is participating. I discussed this problem with my teammates and we spent several days trying to edit the model in order to let it use all the threads of our CPUs until we found out that this has been a problem that is way too hard for us to solve. Even some games which were developed by hundreds of professional programmers, perform poorly in using multiple threads. Wish that someday we would be able to make it.

2020-04-19

海华 AI挑战赛

几个月来一直在参与海华AI挑战赛的培训。这周也终于圆满完成了比赛，虽然我们的团队差0.04分没能进入前三名，但我们也都有很多收获。

一开始上课的时候遇到了很多困难，虽说是面向中学生的AI入门，课程还是涉及到了很多没有接触过的数学知识。课后和组员讨论，额外找了一些网络上的学习资料，几周以后慢慢能理解线性回归、梯度下降之类的理论，也能自己利用开源的模型进行一些训练了。

后期的课程在教安装TensorFlow时提到了CPU和GPU的特性，刚学完时对GPU在图像处理上的优势还没有什么直观的概念。后来比赛时使用CPU版本的TensorFlow运行训练集，10000张耗费的时间超过了十个小时，才尝试改用GPU进行处理。 GPU版本的TensorFlow安装相当复杂，CUDA、 cuDNN两个软件要安装，而且版本要求严苛，第一次安装时cuDNN使用了10.0版本，最后没能运行。第二次安装成功，但运行模型时还是遇到了错误，求助同组的同学后，我降级了电脑使用的Numpy，才正常运行起来。虽然安装过程遇到了许多困难，最后的成果还是令人满意的，GPU运行同样的任务只花了不到一个小时，一天内模型可以迭代多次，这为我们组的分数做出了很大的贡献。删除了数据增强的部分以后，准确率能够达到99.9%以上。但训练六次后，准确率就不再提高了，甚至下降到了98%。我尝试使用另一个预训练的模型，但是没能成功导入，成绩最后也止步于第四名。

虽然没能够获得理想的成绩，但是参与比赛的过程还是收获了很多知识，对我来说是一个十分宝贵的经历。
